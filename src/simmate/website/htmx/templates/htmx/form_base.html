{% comment %} I used to have the base template be whats shown below, but it gave issues when I wanted nested forms. For example, I wanted to have a 'create_many' form where you can edit rows one at a time and *only* send the data for a single row, rather than the whole form on update. When the base component template wraps everything in a form tag, pretty much everything is submitted unless I have unique names for every input row (e.g. name1, name2, name3 for each row). I found it cleanest to just have hx-include whenever necessary so that I have better control over the exact post data.
