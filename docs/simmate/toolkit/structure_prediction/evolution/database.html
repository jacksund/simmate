 13class EvolutionarySearch(DatabaseTable):
 14    """
 15    This database table holds all of the information related to an evolutionary
 16    search and also has convient methods to analyze the data.
 17
 18    Loading Results
 19    ---------------
 20
 21    Typically, you'll load your search through a search id or a composition:
 22
 23    .. code-block:: python
 24
 25        from simmate.shortcuts import SearchResults
 26
 27        # if you know the id
 28        search_results = SearchResults.objects.get(id=123)
 29
 30        # if you know the composition
 31        search_results = SearchResults.objects.get(id="Ca2 N1")
 32
 33    Alternatively, you can find these out by looking at a table of all the
 34    evolutionary searches that have been ran:
 35
 36    .. code-block:: python
 37
 38        all_searches = SearchResults.objects.to_dataframe()
 39
 40    Viewing Results
 41    ---------------
 42
 43    The first thing you may want to check is the best structure found. To access
 44    this and write it to a file:
 45
 46    .. code-block:: python
 47
 48        # loads the best structure and converts it to a pymatgen structure object
 49        structure = search_results.best_individual.to_toolkit()
 50
 51        # writes it to a cif file
 52        structure.to("cif", "best_structure.cif")
 53
 54    To view convergence of the search, you can use the convenient plotting methods.
 55
 56    Note: this will open up the plot in your default browser, so this command
 57    won't work properly through an ssh terminal.
 58
 59    .. code-block:: python
 60
 61        search_results.view_convergence_plot()
 62
 63    If you are benchmarking Simmate to see if it found a particular structure,
 64    you can use:
 65
 66    .. code-block:: python
 67
 68        from simmate.toolkit import Structure
 69
 70        structure = Structure.from_file("example123.cif")
 71
 72        search_results.view_correctness_plot(structure)
 73
 74    Beyond plots, you can also access a table of all calculated structures:
 75
 76    .. code-block:: python
 77
 78        dataframe = search_results.individuals_completed.to_dataframe()
 79

166    def get_correctness_plot(self, structure_known):
167
168        # --------------------------------------------------------
169        # This code is from simmate.toolkit.validators.fingerprint.pcrystalnn
170        # OPTIMIZE: There should be a convience method to make this featurizer
171        # since I use it so much
172        import numpy
173        from simmate.toolkit import Composition
174        from matminer.featurizers.site import CrystalNNFingerprint
175        from matminer.featurizers.structure.sites import PartialsSiteStatsFingerprint
176
177        sitefingerprint_method = CrystalNNFingerprint.from_preset(
178            "ops", distance_cutoffs=None, x_diff_weight=3
179        )
180        featurizer = PartialsSiteStatsFingerprint(
181            sitefingerprint_method,
182            stats=["mean", "std_dev", "minimum", "maximum"],
183        )
184        featurizer.elements_ = numpy.array(
185            [element.symbol for element in Composition(self.composition).elements]
186        )
187        # --------------------------------------------------------
188
189        # Grab the calculation's structure and convert it to a dataframe
190        structures_dataframe = self.individuals_completed.to_dataframe()
191
192        # because we are using the database model, we first want to convert to
193        # pymatgen structures objects and add a column to the dataframe for these
194        #
195        #   structures_dataframe["structure"] = [
196        #       structure.to_toolkit() for structure in ionic_step_structures
197        #   ]
198        #
199        # BUG: the read_frame query creates a new query, so it may be a different
200        # length from ionic_step_structures. For this reason, we can't iterate
201        # through the queryset like in the commented out code above. Instead,
202        # we need to iterate through the dataframe rows.
203        # See https://github.com/chrisdev/django-pandas/issues/138 for issue
204        from simmate.toolkit import Structure
205
206        structures_dataframe["structure"] = [
207            Structure.from_str(s.structure_string, fmt="POSCAR")
208            for _, s in structures_dataframe.iterrows()
209        ]
210
211        from tqdm import tqdm
212
213        structures_dataframe["fingerprint"] = [
214            numpy.array(featurizer.featurize(s.structure))
215            for _, s in tqdm(structures_dataframe.iterrows())
216        ]
217
218        fingerprint_known = numpy.array(featurizer.featurize(structure_known))
219
220        structures_dataframe["fingerprint_distance"] = [
221            numpy.linalg.norm(fingerprint_known - s.fingerprint)
222            for _, s in tqdm(structures_dataframe.iterrows())
223        ]
224
225        # There's only one plot here, no subplot. So we make the scatter
226        # object and just pass it directly to a Figure object
227        scatter = plotly_go.Scatter(
228            x=structures_dataframe["updated_at"],
229            y=structures_dataframe["fingerprint_distance"],
230            mode="markers",
231            marker_color=structures_dataframe["energy_per_atom"],
232        )
233        figure = plotly_go.Figure(data=scatter)
234
235        figure.update_layout(
236            xaxis_title="Date Completed",
237            yaxis_title="Distance from Known Structure",
238        )
239
240        # we return the figure object for the user
241        return figure

244        figure = self.get_correctness_plot(structure_known)
245        figure.show(renderer="browser")

268    def update_flow_run_ids(self):
269
270        # Using our list of current run ids, we query prefect to see which of
271        # these still are running or in the queue.
272        # OPTIMIZE: This may be a really bad way to query Prefect...
273        query = {
274            "query": {
275                with_args(
276                    "flow_run",
277                    {
278                        "where": {
279                            "state": {"_in": ["Running", "Scheduled"]},
280                            "id": {"_in": self.prefect_flow_run_ids},
281                        },
282                    },
283                ): ["id"]
284            }
285        }
286        client = Client()
287        result = client.graphql(query)
288        # graphql gives a weird format, so I reparse it into just a list of ids
289        result = [run["id"] for run in result["data"]["flow_run"]]
290
291        # we now have our new list of IDs! Let's update it to the database
292        self.prefect_flow_run_ids = result
293        self.save()
294
295        # in case we need the list of ids, we return it too
296        return result
297
298    @property
299    def nprefect_flow_runs(self):
300        # update our ids before we report how many there are.
301        runs = self.update_flow_run_ids()
302        # now the currently running ones is just the length of ids!
303        return len(runs)

This database table holds all of the information related to an evolutionary search and also has convient methods to analyze the data.

Loading Results

Typically, you'll load your search through a search id or a composition:

.. code-block:: python

from simmate.shortcuts import SearchResults

# if you know the id
search_results = SearchResults.objects.get(id=123)

# if you know the composition
search_results = SearchResults.objects.get(id="Ca2 N1")

Alternatively, you can find these out by looking at a table of all the evolutionary searches that have been ran:

.. code-block:: python

all_searches = SearchResults.objects.to_dataframe()

Viewing Results

The first thing you may want to check is the best structure found. To access this and write it to a file:

.. code-block:: python

# loads the best structure and converts it to a pymatgen structure object
structure = search_results.best_individual.to_toolkit()

# writes it to a cif file
structure.to("cif", "best_structure.cif")

To view convergence of the search, you can use the convenient plotting methods.

Note: this will open up the plot in your default browser, so this command won't work properly through an ssh terminal.

.. code-block:: python

search_results.view_convergence_plot()

If you are benchmarking Simmate to see if it found a particular structure, you can use:

.. code-block:: python

from simmate.toolkit import Structure

structure = Structure.from_file("example123.cif")

search_results.view_correctness_plot(structure)

Beyond plots, you can also access a table of all calculated structures:

.. code-block:: python

dataframe = search_results.individuals_completed.to_dataframe()

def source(unknown)

Note, this field is highly experimental at the moment and subject to change.

This column indicates where the data came from, and it could be a number of things including...

  • a third party id
  • a structure from a different Simmate datbase table
  • a transformation of another structure
  • a creation method
  • a custom submission by the user

By default, this is a JSON field to account for all scenarios, but some tables (such as those in simmate.database.third_parties) this is value should be the same for ALL entries in the table and therefore the column is overwritten as an attribute.

For other tables where this is not a constant, here are some examples of values used in this column:

source = {

# from a templated structure creator (e.g. substituition or prototypes)
source = {

}

# from a transformation
source = {
    "method": "MirrorMutation",

source = {
